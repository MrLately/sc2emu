# Security Policy

## Supported Versions

Only the latest `main` branch is considered supported for security fixes.

## Reporting a Vulnerability

- Do not post exploit details in public issues.
- Open a private report to project maintainers with:
  - affected commit/version
  - reproduction steps
  - impact assessment
  - proposed mitigation (if available)

Maintainers will acknowledge receipt and coordinate remediation before public disclosure.

## Operational Security Notes

- Do not commit real device identifiers, personal IPs, or private network details.
- Treat diagnostics exports as sensitive; they can include location and operational metadata.
